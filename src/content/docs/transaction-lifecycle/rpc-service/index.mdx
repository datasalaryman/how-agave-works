title: RPC Service
---

The RPC service is the main interface between users and services and the
validator client.

## JSON-RPC

Agave's RPC service is an HTTP server like most RPC nodes (e.g. Helius, Triton
One, etc) with two differences:

- It is tied to the internal state of the Agave client. If it diverges from the
  network or is delayed, you will get those results in your HTTP responses.
- It has a specialized Send Transaction Service that sends the transactions to
  the TPU by batch.

You can view the full specifications of the RPC service's HTTP methods
[here](https://solana.com/docs/rpc).

## Transaction checking and sending

The **RPC Service** converts the transaction data into bytes that the TPU can
consume. It does several checks and makes sure that it can be sent.

The data is handed off to the **Send Transaction Service**, whose purpose is to
handle transaction sending, and retry logic.

The last piece is the **tpu-client-next** crate, which handles actually sending
the bytes to the TPU.
